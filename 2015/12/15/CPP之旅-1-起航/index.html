命名空间

命名空间对全局名字空间进行再次划分，避免在大型项目中，使用的类库中有重名的全局函数或者全局变量的机制。

global scope 是一个程序中最大的scope，也是引起命名冲突的根源，C语言没有从语言的机制上解决这个问题。

语法

使用名字空间内的内容时，使用域作用符(::)进行访问。

using namespace 名字空间;
可以在之后使用名字空间内容时不需要带上名字空间。

输入与输出

cout与cin是cpp中的标准输入流和标准输出流，他们定义在头文件iostream中(注意:在新版cpp中头文件都是不带.h的)，他们属于名字空间std。

函数重载

构成重载规则

1.在同一个名字空间
2.函数名相同。
3.参数个数不同，参数的类型不同，参数顺序不同，均可构成重载。
4.返回值类型不同则不可以构成重载。

匹配原则

1.严格匹配，找到则调用。
2.通过隐式转换寻求一个匹配，找到则调用。

注:cpp 允许，int 到long 和double 隐式类型转换。所以在做函数重载时要注意二义性的发生。

底层实现

cpp使用name mangling(倾轧)技术，来改名函数名，区分参数不同的同名函数。

规则

1.默认的顺序，是从右向左，不能跳跃。
2.定义在前，调用在后(此时定义和声明为一体)，默认认参数在定义处。声明在前，调用在后，默认参数在声明处。
3.一个函数，不能既作重载，又作默认参数的函数。当你少写一个参数时，系统无法确认是重载还是默认参数。

引用

引用的本质是指针，C++对裸露的内存地址(指针)作了一次包装。又取得的指针的优良
特性。所以再对引用取地址，建立引用的指针没有意义。
1.可以定义指针的引用，但不能定义引用的引用。
2.可以定义指针的指针(二级指针)，但不能定义引用的指针。
3.可以定义指针数组，但不能定义引用数组，可以定义数组引用。
4,常引用
const 引用有较多使用。它可以防止对象的值被随意修改。因而具有一些特性。
（1）const 对象的引用必须是const 的，将普通引用绑定到const 对象是不合法的。
这个原因比较简单。既然对象是const 的，表示不能被修改，引用当然也不能修改，必
须使用const 引用。
（2）const 引用可使用相关类型的对象(常量,非同类型的变量或表达式)初始化。
这个是const 引用与普通引用最大的区别。const int &a=2;是合法的。double x=3.14; const

常引用原理：
const 引用的目的是，禁止通过修改引用值来改变被引用的对象。const 引用的初始
化特性较为微妙.

内联函数

c 语言中有宏函数的概念。宏函数的特点是内嵌到调用代码中去，避免了函数调用的开
销。但是由于宏函数的处理发生在预处理阶段，缺失了语法检测和有可能带来的语意差错。

语法

cpp中提供inline关键字来实现内联函数，inline只是向编译器申请实现内联，至于是否内联需要编译器决定。

原则

1.内内联函数需要短小精悍
2.内联函数不要有循环
3.内联函数不要有太多的判断语句

评价

优点：避免调用时的额外开销（入栈与出栈操作）
代价：由于内联函数的函数体在代码段中会出现多个“副本”，因此会增加代码段的空间。
本质：以牺牲代码段空间为代价，提高程序的运行时间的效率。
适用场景：函数体很“小”，且被“频繁”调用
